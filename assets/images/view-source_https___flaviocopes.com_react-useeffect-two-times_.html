<p>React 18 released in March 2022 changed the default behavior of <code>useEffect()</code>.</p>

<p>I didn&rsquo;t even realize it at first, reading the React 18 launch post, buried along a lot of new features.</p>

<p>If your application is behaving strangely after updating to React 18, the default behavior of <code>useEffect</code> changed to run it <strong>2 times</strong>.</p>

<p><strong>Just in development mode</strong>, but this is the mode everyone builds their application on.</p>

<p>And just in <strong>strict mode</strong>, but that&rsquo;s now the default for applications built using <code>create-react-app</code> or Next.js.</p>

<p>There are reasons for this.</p>

<p>Its not a problem of your code - its how things work now in React.</p>

<p>The only way to disable this behavior is to <strong>disable <a href="https://reactjs.org/docs/strict-mode.html">strict mode</a></strong>.</p>

<p>Strict mode is important so this is a temporary workaround until you can fix any issue this change introduced.</p>

<p>You can do it in Next.js by adding the option</p>
